# paraphrasing-korean — 한국어 문장 다듬기

한국어 글을 **뜻은 한 글자도 바꾸지 않고** 문장만 깔끔하게 다듬어 주는 AI 에이전트 스킬입니다.
글을 통째로 고치는 게 아니라, **문장 하나하나를 따로 떼어 읽고** 군더더기만 덜어 냅니다.

세 가지를 손봅니다. **번역투**(영어를 직역한 듯한 어색한 말투), **AI 티**(챗봇이 쓴 듯한 기계적인 말투),
그리고 **문장 다이어트**(없어도 되는 군더더기 빼기)입니다. 문장 다이어트 규칙은 책
『내 문장이 그렇게 이상한가요?』에서 영감을 받았습니다.

```
바꾸기 전:  데이터 분석을 통해 얻어진 결과는 매우 의미하는 바가 크다고 할 수 있다.
바꾼 뒤:    데이터를 분석해 얻은 결과는 매우 의미가 크다.
```

## 어떤 명령어가 있나요

네 가지입니다. 하나만 쓰고 싶으면 따로, 한꺼번에 하고 싶으면 통합 명령을 씁니다.

| 명령어 | 하는 일 |
|---|---|
| `/paraphrasing-korean` | **통합** — 번역투 → AI 티 → 문장 다이어트를 한 문장씩 차례로 다 적용 |
| `/paraphrasing-korean-센텐스` (영문 별칭 `-sentence`) | 문장 다이어트 (적·의·것·들, 군더더기 "있다", 군더더기 피동, 조사 다듬기) |
| `/paraphrasing-korean-trans` | 번역투 빼기 (피동→능동, 가지다·통해·대해, 무생물 주어, 대명사 남용) |
| `/paraphrasing-korean-ai` | AI 티 빼기 (첫째·둘째 나열, "결국·시사하는 바가 크다", 과장 어휘, 이모지) |

설치하면 명령어로 부르거나, 그냥 "이 글 문장 좀 다듬어줘" 처럼 말해도 알아서 골라 줍니다.

## 설치

여러 AI 도구(Claude Code · Codex · OpenClaw · Hermes)에서 그대로 씁니다. 같은 스킬 파일을 각 도구의
폴더에 연결해 주는 설치 스크립트가 있습니다.

```bash
git clone https://github.com/Turtle-Hwan/paraphrasing-korean
cd paraphrasing-korean

./install.sh claude      # Claude Code
./install.sh codex       # Codex
./install.sh openclaw    # OpenClaw
./install.sh hermes      # Hermes
./install.sh all         # 깔려 있는 도구 전부 한 번에
```

기본은 **연결(심링크)**이라 `git pull` 만 하면 최신으로 갱신됩니다. 복사로 깔고 싶으면 뒤에 `--copy` 를 붙입니다.

### Claude Code 플러그인으로도 쓸 수 있어요

이 저장소에는 플러그인 설정 파일(`.claude-plugin/plugin.json`)이 들어 있어, 네 명령어를 한 번에 묶어
플러그인으로 쓸 수 있습니다. 로컬에서 확인: `claude plugin validate . --strict`.

## 어떻게 동작하나요 — 한 문장씩 다듬기

핵심은 **"문장 단위로 다 넣고 빼서 다듬는다"** 입니다. 정규식으로 한꺼번에 바꾸지 않습니다.
한 문장이라도 빠뜨리지 않으려고, 작은 도구 두 개로 흐름을 묶어 두었습니다.

1. **자르기** (`scripts/segment.py`) — 입력을 마침표·물음표 같은 문장 부호 기준으로 잘라
   문장마다 한 칸씩 만든 작업표(`worksheet.md`)를 뽑습니다. 제목·목록·코드 블록 같은 건 "그대로 둘 줄"로
   표시해 손대지 않습니다.
2. **다듬기** — 에이전트가 작업표의 **문장 칸을 위에서 아래로 하나씩 읽고** 다듬어 채웁니다.
   규칙(아래)을 근거로 삼고, 고칠 게 없으면 원문 그대로 둡니다.
3. **되붙이기** (`scripts/reassemble.py`) — 채운 작업표를 원래 구조에 맞춰 다시 한 편의 글로 잇습니다.
   이때 두 가지를 자동으로 지킵니다.
   - **문장 수 검사**: 들어간 문장 수와 나온 문장 수가 다르면 멈춥니다(문장이 사라지거나 합쳐지는 걸 막음).
   - **변경량 검사**: 너무 많이 바뀌면(절반 넘게) 멈춥니다(원문을 망칠 만큼 고치는 걸 막음).

원하면 중간에 **정규식 힌트**(`scripts/scan.py`)를 켜도 됩니다. 의심 가는 문장에 규칙 번호를 살짝
표시해 줄 뿐, 다듬는 건 어디까지나 에이전트가 직접 합니다(힌트는 틀릴 수도 있어요).

## 규칙은 어디서 왔나요

규칙은 모두 `references/` 폴더에 글로 정리해 두었습니다.

- `sentence-rules.md` — 문장 다이어트. 책 『내 문장이 그렇게 이상한가요?』(유유, 2016)에서 영감을 받았습니다 —
  [교보문고 도서 페이지](https://product.kyobobook.co.kr/detail/S000001863138).
- `translationese-rules.md` — 번역투. 영어 직역 흔적을 자연스러운 한국어로 바꾸는 규칙.
- `ai-tell-rules.md` — AI 티. 챗봇 특유의 말투·구조·리듬을 손보는 규칙.
- `prime-directives.md` — 모든 명령에 공통으로 지키는 약속(뜻 보존, 안 건드리는 것, 변경량 한계, 자체검증).

각 규칙은 "무엇을 찾고, 어떻게 고치고, 예를 들면 이렇게" 형식이라, 예시만 읽어도 감이 옵니다.
