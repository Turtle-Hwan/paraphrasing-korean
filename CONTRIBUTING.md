# 기여

예시 하나만 보태도 도움이 돼요. 어려운 코드 없이 규칙과 예시만 늘려도 돼요.

## 보탤 수 있는 것

1. **예시 추가** — 규칙은 맞는데 예시가 적으면 `바꾸기 전 → 바꾼 뒤` 예시를 더 넣어 주세요.
2. **새 규칙** — 자주 보이는 맞춤법·문체·번역 문체·AI 문체·군더더기인데 아직 규칙에 없으면 새로 적어 주세요.

## 규칙 적는 법

규칙 글은 `references/` 안에 있어요. 종류별로 파일이 나뉘어 있어요.

- 맞춤법·문체 → `grammar-rules.md` · `style-guide.md`
- AI 문체 → `ai-tell-rules.md`
- 문장 교정(번역투·군더더기) → `sentence-rules.md`
- 근거 지도 → `sources.md`

규칙 하나는 이렇게 생겼어요. 그대로 따라 적으면 돼요.

```markdown
## S-27. 규칙 이름 [S1]

무엇을 찾는지 한 줄로 적는다.

- 바꾸기 전 문장 → 바꾼 뒤 문장
- 바꾸기 전 문장 → 바꾼 뒤 문장
```

- 번호는 종류별 머리글자에 다음 숫자를 붙여요. 맞춤법 `G-`, 문장(번역투·군더더기) `S-`, AI 문체 `AI-`, 문체 `ST-`.
- `[S1]` 은 "거의 항상 고친다", `[S2]` 는 "문맥 보고 고친다"는 뜻이에요.
- **새 규칙보다 예시가 먼저예요.** 새 패턴은 기존 규칙의 예시로 흡수할 수 있는지 먼저 보고, 못 담을 때만 새 번호를 만들어요. 규칙은 적게, 예시는 넉넉히 — 그래야 룰북이 가벼워요.
- 국립국어원 자료를 근거로 맞춤법·문체 규칙을 제안할 때는 `references/sources.md` 도 함께 봐 주세요.

> [!IMPORTANT]
> **뜻을 바꾸는 규칙은 받지 않아요.** 이 도구는 뜻은 그대로 두고 표현만 다듬어요.

## 제출 전 점검

`scripts/` 코드를 고쳤다면 검사를 한 번 돌려 주세요. 파이썬 표준 기능만 쓰니 따로 설치할 게 없어요.
`python3` 가 없으면 `python`(Windows 는 `py -3`)으로 실행해요.

```bash
python3 -m unittest discover -s tests
```

`references/` 나 `scripts/` 를 고쳤다면 각 스킬 패키지에 사본을 맞추도록 동기화를 돌려 주세요.
(안 맞추면 위 검사가 실패해요. 심링크가 Windows 에서 깨져서 실제 파일로 복사해 두거든요.)

```bash
python3 scripts/sync_skills.py
```

규칙과 예시만 고쳤더라도 위 동기화는 돌려 주세요. 그 밖엔 편하게 Pull Request 를 열어 주세요.
