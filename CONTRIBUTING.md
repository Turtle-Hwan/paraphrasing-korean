# 같이 만들기

예시 하나만 보태도 도움이 됩니다. 어려운 코드 없이, 글(규칙)과 예시만 늘려도 됩니다.

## 무엇을 보탤 수 있나요

1. **예시 추가** — 규칙은 맞는데 예시가 적을 때, `바꾸기 전 → 바꾼 뒤` 예시를 더 넣어 주세요.
2. **새 규칙** — 자주 보이는 번역투·AI 티·군더더기인데 아직 규칙에 없으면 새로 적어 주세요.
3. **정규식 힌트** — 정규식으로 쉽게 잡히는 표현이면 힌트를 추가해 주세요(아래).

## 규칙은 어떻게 적나요

규칙 글은 `references/` 안에 있습니다. 종류별로 파일이 나뉘어 있어요.

- 번역투 → `translationese-rules.md`
- AI 티 → `ai-tell-rules.md`
- 문장 다이어트 → `sentence-rules.md`

규칙 하나는 이렇게 생겼습니다. 그대로 따라 적으면 됩니다.

```markdown
## T-10. 규칙 이름 [S1]

무엇을 찾는지 한 줄로 적는다.

- 바꾸기 전 문장 → 바꾼 뒤 문장
- 바꾸기 전 문장 → 바꾼 뒤 문장
```

- 번호는 종류별 머리글자 + 다음 숫자입니다. 번역투 `T-`, AI 티 `AI-`, 문장 다이어트 `S-`.
- `[S1]` 은 "거의 항상 고친다", `[S2]` 는 "문맥 보고 고친다" 라는 뜻입니다.
- **뜻을 바꾸는 규칙은 받지 않습니다.** 이 도구는 뜻은 그대로 두고 표현만 다듬습니다.

## 정규식 힌트 추가하기

정규식으로 쉽게 잡히는 표현은 `references/regex-patterns.json` 에 한 줄 추가하면 됩니다.

```json
{ "id": "T-10", "name": "규칙 이름", "rulebook": "trans", "regex": "찾을\\s*표현" }
```

- `rulebook` 은 `sentence` / `trans` / `ai` 중 하나입니다.
- 정규식은 **힌트일 뿐** 입니다. 틀려도(엉뚱한 문장을 잡아도) 큰 문제는 없습니다. 다듬는 건 사람·에이전트가 하니까요.
- 한국어는 조사가 자주 바뀌니 너무 빡빡하게 잡지 말고 느슨하게 적어 주세요.

## 보내기 전에

코드(`scripts/`)를 고쳤다면 검사를 한 번 돌려 주세요. 파이썬 표준 기능만 쓰므로 따로 깔 게 없습니다.

```bash
python3 -m unittest discover -s tests
```

규칙·예시만 고쳤다면 검사는 안 돌려도 됩니다. 편하게 Pull Request 를 열어 주세요.

## 작은 약속

- 한 번에 한 가지만 (예시 묶음 하나, 규칙 하나) 보내면 검토가 빠릅니다.
- 예시는 가능하면 실제로 본 어색한 문장으로 부탁드립니다.
- 책 『내 문장이 그렇게 이상한가요?』의 문장을 그대로 옮겨 싣지는 말아 주세요(영감은 OK, 복제는 NO).
